# MCP Server Configuration

Dieses Projekt enthält den **n8n-ai-agents** MCP Server als Git-Submodul unter `mcp-servers/n8n-mcp-tools/`.

## Setup

### 1. Dependencies installieren

```bash
cd mcp-servers/n8n-mcp-tools
npm install
```

### 2. Claude Code konfigurieren

Die Datei `.claude/settings.local.json` im Projektroot enthält bereits die MCP-Konfiguration:

```json
{
  "mcpServers": {
    "n8n-ai-agents": {
      "command": "node",
      "args": ["/ABSOLUTER/PFAD/ZU/n8n-ai-agents/mcp-servers/n8n-mcp-tools/server.js"]
    }
  }
}
```

> **Hinweis:** `settings.local.json` ist gitignored, da sie einen absoluten Pfad enthält. Jeder Nutzer muss den Pfad an sein System anpassen.

### 3. Testen

```bash
cd mcp-servers/n8n-mcp-tools
node server.js
```

Erwartete Ausgabe: `n8n AI Agents MCP Server running on stdio` (Ctrl+C zum Stoppen).

Nach einem Neustart von Claude Code stehen die MCP-Tools automatisch zur Verfügung.

## Verfügbare Tools

| Tool | Beschreibung |
|------|-------------|
| `get_n8n_best_practices` | Best Practices für robuste n8n-Workflows |
| `generate_workflow_template` | Workflow-Templates generieren |
| `validate_workflow_json` | Workflow-JSON gegen Best Practices validieren |
| `get_prompt_template` | Optimierte Prompt-Templates für kleine LLMs |
| `analyze_test_results` | Test-Ergebnisse aus n8n-Workflows analysieren |
| `find_native_node` | Prüfen ob n8n einen nativen Node für einen Service hat |
